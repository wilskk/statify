# Desain Fitur: Definisi Tipe Domain

Dokumen ini berisi paket desain (Design Package) untuk Feature Set `Definisi Tipe Domain`.

---

## 2. Design Package

### 2.1. Diagram Urutan (Sequence Diagrams)

*Karena fitur ini berfokus pada definisi tipe statis, diagram kelas atau diagram komponen lebih relevan daripada diagram urutan. Diagram berikut mengilustrasikan hubungan antar tipe domain utama.*

```mermaid
classDiagram
    title "Struktur Tipe Domain Inti"

    class Data {
      <<interface>>
    }
    class Variable {
      <<interface>>
    }
    class Result {
      <<interface>>
    }
    class Meta {
      <<interface>>
    }

    Variable "1" -- "0..*" Data : "defines structure of"
    Result "0..*" -- "1" Meta : "is part of"
```

### 2.2. Penyempurnaan Model Objek (Object Model Refinements)

*Perubahan pada model objek (kelas, atribut, metode baru) yang ditemukan selama desain.*

- **Tipe Baru:**
  - `types/Data.ts`: Mendefinisikan struktur untuk data tabular.
  - `types/Variable.ts`: Mendefinisikan properti dari sebuah variabel (misal: nama, tipe, label).
  - `types/Result.ts`: Mendefinisikan struktur untuk output hasil analisis.
  - `types/Meta.ts`: Mendefinisikan metadata level aplikasi (misal: nama file).
  - `types/modalTypes.ts`: Mendefinisikan tipe yang terkait dengan sistem modal (misal: ID modal, props).

### 2.3. Catatan Alternatif Desain (Design Alternatives)

*Diskusi dan keputusan mengenai pilihan desain yang signifikan.*

- **Alternatif 1:** Menggunakan `class` JavaScript daripada `interface` TypeScript.
  - **Kelebihan:** Dapat menyertakan metode dan logika bisnis di dalam model.
  - **Kekurangan:** Akan mengarah pada perancangan yang lebih kompleks (OOP) di mana cukup dengan struktur data sederhana. Pola yang digunakan di aplikasi ini memisahkan data (tipe), state (store), dan logika (service), sehingga `interface` lebih sesuai.
- **Keputusan:** Menggunakan `interface` atau `type` TypeScript untuk menjaga agar definisi domain tetap sederhana sebagai kontrak data, sejalan dengan arsitektur fungsional dan pemisahan concern.
