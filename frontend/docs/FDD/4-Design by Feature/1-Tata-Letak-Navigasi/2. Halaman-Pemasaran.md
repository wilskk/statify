# Desain Fitur: Halaman Pemasaran

Dokumen ini berisi paket desain (Design Package) untuk Feature Set `Halaman Pemasaran`.

---

## 2. Design Package

### 2.1. Diagram Urutan (Sequence Diagrams)

Diagram urutan berikut menunjukkan bagaimana komponen utama halaman pemasaran dirender secara berurutan untuk membangun halaman landing.

```mermaid
sequenceDiagram
    participant User as Pengguna
    
    box "Aplikasi Frontend"
        participant Page as "LandingPage"
        participant Header as "LandingHeader"
        participant Hero as "HeroSection"
        participant Features as "FeaturesSection"
        participant Comparison as "ComparisonSection"
        participant Testimonials as "TestimonialsSection"
        participant Footer as "LandingFooter"
    end

    autonumber

    Pengguna->>Page: Mengakses URL halaman landing
    activate Page

    Note right of Page: Halaman me-render<br/>semua komponen anaknya.

    Page->>Header: Render Header
    Page->>Hero: Render Bagian Hero
    Page->>Features: Render Bagian Fitur
    Page->>Comparison: Render Bagian Perbandingan
    Page->>Testimonials: Render Bagian Testimoni
    Page->>Footer: Render Footer
    
    deactivate Page
end
```

### 2.2. Penyempurnaan Model Objek (Object Model Refinements)

*Perubahan pada model objek (kelas, atribut, metode baru) yang ditemukan selama desain.*

- **Kelas Baru:**
  - `LandingHeader.tsx`: Komponen header khusus untuk halaman landing.
  - `HeroSection.tsx`: Komponen untuk bagian "hero" yang berisi judul utama dan ajakan bertindak (call-to-action).
  - `FeaturesSection.tsx`: Komponen untuk menampilkan fitur-fitur utama aplikasi.
  - `ComparisonSection.tsx`: Komponen untuk membandingkan Statify dengan aplikasi sejenis.
  - `TestimonialsSection.tsx`: Komponen untuk menampilkan testimoni dari pengguna.
  - `LandingFooter.tsx`: Komponen footer khusus untuk halaman landing.
- **Atribut/Metode Baru:**
  - Tidak ada.

### 2.3. Catatan Alternatif Desain (Design Alternatives)

*Diskusi dan keputusan mengenai pilihan desain yang signifikan.*

- **Alternatif 1:** Menggunakan library animasi kompleks (misal: Framer Motion) untuk transisi antar seksi.
  - **Kelebihan:** Pengalaman pengguna bisa menjadi lebih dinamis dan menarik secara visual.
  - **Kekurangan:** Menambah ukuran bundel JavaScript, meningkatkan beban loading halaman, dan menambah kompleksitas implementasi serta pemeliharaan.
- **Keputusan:** Menggunakan transisi CSS sederhana yang dioptimalkan oleh `Tailwind CSS`. Keputusan ini memprioritaskan performa halaman (kecepatan loading) dan kemudahan pengembangan, sambil tetap memberikan efek visual yang halus dan profesional.
