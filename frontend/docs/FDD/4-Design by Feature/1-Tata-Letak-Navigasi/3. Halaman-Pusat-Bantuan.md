# Desain Fitur: Halaman Pusat Bantuan

Dokumen ini berisi paket desain (Design Package) untuk Feature Set `Halaman Pusat Bantuan`.

---

## 2. Design Package

### 2.1. Diagram Urutan (Sequence Diagrams)

Diagram urutan berikut menunjukkan bagaimana pengguna berinteraksi dengan Papan Bantuan untuk menavigasi antar seksi dan menampilkan konten yang relevan.

```mermaid
sequenceDiagram
    actor User as "Pengguna"
    
    box "Aplikasi Frontend"
        participant Page as "HelpPage"
        participant Content as "HelpContent"
        participant Sidebar as "Sidebar"
        participant Main as "Area Konten"
    end

    autonumber

    Pengguna->>Page: Mengakses halaman bantuan
    activate Page

    Page->>Content: Render
    activate Content
    
    Content->>Sidebar: Render item navigasi
    Content->>Main: Render konten default (misal: Getting Started)

    Pengguna->>Sidebar: Klik item navigasi (misal: "FAQ")
    
    Sidebar-->>Page: Memicu callback onSectionSelect('faq')
    
    Note right of Page: Update state untuk section aktif
    
    Page->>Content: Re-render dengan section aktif
    Content->>Main: Render komponen konten yang sesuai (misal: FAQ.tsx)
    
    deactivate Content
    deactivate Page
end
```

### 2.2. Penyempurnaan Model Objek (Object Model Refinements)

*Perubahan pada model objek (kelas, atribut, metode baru) yang ditemukan selama desain.*

- **Kelas Baru:**
  - `HelpContent.tsx`: Komponen utama yang mengatur tata letak halaman bantuan, termasuk sidebar navigasi dan area konten utama.
  - `GettingStarted.tsx`: Komponen yang berisi panduan langkah pertama untuk menggunakan aplikasi.
  - `FAQ.tsx`: Komponen yang menampilkan daftar pertanyaan yang sering diajukan beserta jawabannya.
  - `Feedback.tsx`: Komponen yang menyediakan formulir bagi pengguna untuk mengirimkan masukan atau laporan bug.
  - `StatisticsGuide.tsx`: Komponen yang berfungsi sebagai router untuk menampilkan panduan statistik yang spesifik.
  - `guide-content/*.tsx`: Berbagai komponen yang berisi konten untuk setiap panduan individual (misal: `Frequencies.tsx`, `Descriptives.tsx`).
- **Atribut/Metode Baru:**
  - Tidak ada.

### 2.3. Catatan Alternatif Desain (Design Alternatives)

*Diskusi dan keputusan mengenai pilihan desain yang signifikan.*

- **Alternatif 1:** Menampilkan semua konten bantuan dalam satu halaman panjang yang dapat di-scroll.
  - **Kelebihan:** Implementasi paling sederhana, tidak memerlukan manajemen state untuk navigasi.
  - **Kekurangan:** Sulit bagi pengguna untuk menemukan informasi spesifik dengan cepat. Pengalaman pengguna menjadi kurang baik seiring bertambahnya konten.
- **Keputusan:** Menggunakan layout dengan sidebar navigasi. Pendekatan ini memisahkan konten ke dalam beberapa bagian yang logis (`Getting Started`, `FAQ`, `Feedback`, `Guides`), membuatnya lebih terstruktur, mudah diakses, dan memberikan pengalaman pengguna yang lebih baik.
