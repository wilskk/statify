# Desain Fitur: Halaman Dashboard

Dokumen ini berisi paket desain (Design Package) untuk Feature Set `Halaman Dashboard`.

---

## 2. Design Package

### 2.1. Diagram Urutan (Sequence Diagrams)

Diagram berikut mengilustrasikan bagaimana komponen-komponen utama yang membentuk tata letak dashboard dirakit saat pengguna mengakses ruang kerja. Diagram ini telah disesuaikan untuk mengikuti praktik terbaik yang diuraikan dalam panduan visualisasi.

```mermaid
sequenceDiagram
    autonumber
    actor User as "Pengguna"

    box "Komponen Tata Letak (Layout Components)"
        participant Layout as "DashboardLayout.tsx"
        participant Header as "Header.tsx"
        participant Navbar as "Navbar.tsx"
        participant Toolbar as "Toolbar.tsx"
        participant Workspace as "Workspace (children)"
        participant Footer as "Footer.tsx"
    end

    User->>Layout: 1. Mengakses halaman dashboard
    activate Layout

    Layout->>Header: 2. Render Header
    Layout->>Navbar: 3. Render Navbar
    Layout->>Toolbar: 4. Render Toolbar
    Layout->>Workspace: 5. Render konten aktif (Data, Variabel, atau Hasil)
    Layout->>Footer: 6. Render Footer

    Layout-->>User: 7. Tampilkan halaman dashboard lengkap
    deactivate Layout
end
```

### 2.2. Penyempurnaan Model Objek (Object Model Refinements)

*Perubahan pada model objek (kelas, atribut, metode baru) yang ditemukan selama desain.*

- **Kelas Baru:**
  - `DashboardLayout.tsx`: Komponen utama yang mengatur struktur layout.
  - `Header.tsx`: Komponen untuk bagian header aplikasi.
  - `Navbar.tsx`: Komponen untuk menu navigasi utama (File, Edit, View, dll.).
  - `Toolbar.tsx`: Komponen untuk tombol aksi cepat.
  - `Footer.tsx`: Komponen untuk bagian footer aplikasi.
  - `DashboardLanding.tsx`: Komponen yang ditampilkan di area kerja saat tidak ada data yang dimuat.

- **Atribut/Metode Baru:**
  - Tidak ada penambahan signifikan pada level ini.

### 2.3. Catatan Alternatif Desain (Design Alternatives)

*Diskusi dan keputusan mengenai pilihan desain yang signifikan.*

- **Alternatif 1:** Menggabungkan Navbar dan Toolbar menjadi satu komponen header tunggal.
  - **Kelebihan:** Mengurangi kompleksitas visual dan menyatukan semua aksi di satu tempat.
  - **Kekurangan:** Dapat membatasi ruang yang tersedia untuk tombol aksi cepat (Toolbar) dan membuat menu utama (Navbar) terasa lebih padat.
- **Keputusan:** Memisahkan `Navbar` untuk navigasi dan fungsi global, dan `Toolbar` untuk aksi kontekstual yang sering digunakan. Pendekatan ini memberikan struktur antarmuka yang lebih jelas, skalabel, dan familiar bagi pengguna aplikasi desktop.
