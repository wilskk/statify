## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi logika pemrosesan file Excel dan alur kerja wizard.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Mocking Dependencies:** `DataService` dan library parsing (`SheetJS/xlsx`) di-mock.
  - **Pengujian Logika Parsing:** Memvalidasi bahwa setelah file dipilih, fungsi parsing dari `SheetJS` dipanggil dan daftar nama sheet diekstrak dengan benar.
  - **Pengujian Logika Konfigurasi:** Memastikan bahwa logika untuk mengekstrak data dari sheet dan rentang sel yang dipilih berfungsi sesuai harapan.
  - **Pengujian Interaksi:** Memvalidasi bahwa memilih sheet yang berbeda akan memperbarui pratinjau data.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `ImportExcelSelectionStep` | Pilih file Excel mock | Fungsi parsing dipanggil dan wizard maju ke langkah berikutnya | `Lulus` |
| `ImportExcelConfigurationStep` | Parsing mengembalikan 3 nama sheet | Dropdown sheet berisi 3 opsi | `Lulus` |
| `ImportExcelConfigurationStep` | Pilih sheet yang berbeda | Pratinjau data diperbarui untuk menampilkan data dari sheet terpilih | `Lulus` |
| `ImportExcelConfigurationStep` | Klik 'Selesai' | `DataService.loadNewData` dipanggil dengan data yang diekstrak | `Lulus` |

*Catatan: Pengujian memastikan bahwa file Excel dapat dibaca dan data dapat diekstrak secara akurat berdasarkan pilihan pengguna.*
