## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi logika pemrosesan dan alur kerja wizard impor dari clipboard.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Mocking Dependencies:** `DataService` dan library parsing (`PapaParse`) di-mock.
  - **Pengujian Logika (Processor):** Memvalidasi `useImportClipboardProcessor` memanggil fungsi parsing dengan konfigurasi yang benar (misalnya, delimiter tab) dan kemudian memanggil `DataService` dengan data yang sudah di-parse.
  - **Pengujian State (Logic):** Memastikan `useImportClipboardLogic` mengelola state wizard (langkah saat ini, data mentah) dengan benar.
  - **Pengujian Komponen:** Memverifikasi bahwa setiap langkah wizard di-render dengan benar.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useImportClipboardLogic` | Panggil `handleNextStep` | State `step` diperbarui ke 'Configuration' | `Lulus` |
| `useImportClipboardProcessor` | Panggil `processImport` dengan konfigurasi | `Papa.parse` dipanggil dengan konfigurasi tersebut | `Lulus` |
| `useImportClipboardProcessor` | Parsing berhasil | `DataService.loadNewData` dipanggil | `Lulus` |
| `ImportClipboardConfigurationStep` | Render dengan data pratinjau | Tabel pratinjau menampilkan data | `Lulus` |

*Catatan: Pengujian memastikan bahwa data dari clipboard dapat diproses dengan benar sesuai konfigurasi pengguna dan dimuat ke dalam aplikasi.*
