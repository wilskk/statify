## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi logika pemrosesan file CSV dan alur kerja wizard.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Mocking Dependencies:** `DataService` dan library parsing (`PapaParse`) di-mock.
  - **Pengujian Logika (Processor):** Memvalidasi `useImportCsvProcessor` memanggil fungsi parsing dengan file dan konfigurasi yang benar, lalu memanggil `DataService` dengan data hasil parse.
  - **Pengujian State (Wizard):** Memastikan komponen `ImportCsv/index.tsx` berhasil mengelola transisi antar langkah (dari pemilihan ke konfigurasi).
  - **Pengujian Komponen:** Memverifikasi bahwa pratinjau data di `ImportCsvConfiguration` me-render ulang saat opsi parsing diubah.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `ImportCsv/index.tsx` | Pilih file | Wizard berpindah ke langkah konfigurasi | `Lulus` |
| `useImportCsvProcessor` | Panggil `processImport` | `Papa.parse` dipanggil dengan objek file dan konfigurasi | `Lulus` |
| `useImportCsvProcessor` | Parsing berhasil | `DataService.loadNewData` dipanggil | `Lulus` |
| `ImportCsvConfiguration` | Ubah delimiter | Fungsi parsing dipanggil ulang untuk memperbarui pratinjau | `Lulus` |

*Catatan: Pengujian memastikan bahwa file CSV dapat diproses dengan benar sesuai konfigurasi pengguna dan dimuat ke dalam aplikasi.*
