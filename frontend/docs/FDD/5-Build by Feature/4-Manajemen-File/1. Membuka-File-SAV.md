## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi logika yang terkandung dalam hook dan interaksi komponen.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Mocking Dependencies:** `ClientSavService` dan `DataService` di-mock untuk mengisolasi logika hook `useOpenSavFileLogic`.
  - **Pengujian Logika (Hook):** Memvalidasi bahwa hook memanggil `ClientSavService.uploadFile` saat file dipilih, dan kemudian memanggil `DataService.loadNewData` dengan data yang dikembalikan oleh service API.
  - **Pengujian Interaksi Komponen:** Memastikan komponen `OpenSavFileModal` memanggil `handleFileSelect` dari hook saat pengguna memilih file.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useOpenSavFileLogic` | Panggil `handleFileSelect` dengan file mock | `ClientSavService.uploadFile` dipanggil dengan file mock | `Lulus` |
| `useOpenSavFileLogic` | `uploadFile` berhasil | `DataService.loadNewData` dipanggil dengan data hasil | `Lulus` |
| `useOpenSavFileLogic` | `uploadFile` gagal | State `error` di dalam hook diatur dengan pesan error | `Lulus` |
| `OpenSavFileModal` | Pengguna memilih file | Fungsi `handleFileSelect` dari hook dipanggil | `Lulus` |

*Catatan: Pengujian memastikan alur kerja dari interaksi pengguna hingga pembaruan data aplikasi berjalan dengan benar, dengan dependensi eksternal diisolasi.*
