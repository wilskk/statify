## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi logika pembuatan string CSV.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`
- **Fokus Pengujian:**
  - **Mocking Dependencies:** `useDataStore`, `useMetaStore`, dan library `PapaParse` di-mock.
  - **Pengujian Logika (Hook):** Memvalidasi `useExportCsv` mengambil data dari store, memformatnya jika perlu, dan memanggil `Papa.unparse` dengan data dan konfigurasi yang benar.
  - **Pengujian Fungsionalitas:** Memastikan logika untuk memicu unduhan (membuat Blob dan URL objek) dipanggil setelah string CSV dihasilkan.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useExportCsv` | Panggil `handleExport` | Data diambil dari `useDataStore` dan `useMetaStore` | `Lulus` |
| `useExportCsv` | Panggil `handleExport` dengan konfigurasi | `Papa.unparse` dipanggil dengan data dan konfigurasi yang benar | `Lulus` |
| `useExportCsv` | `Papa.unparse` mengembalikan string CSV | Logika untuk membuat Blob dan memicu unduhan dijalankan | `Lulus` |

*Catatan: Pengujian memastikan bahwa data aplikasi dapat diubah menjadi format CSV dengan benar sesuai dengan opsi yang dipilih pengguna.*
