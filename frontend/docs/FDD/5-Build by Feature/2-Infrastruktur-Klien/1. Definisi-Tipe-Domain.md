## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `TypeScript Compiler (tsc)`
- **Fokus Pengujian:**
  - **Validasi Statis:** Karena fitur ini murni tentang definisi tipe (`interface` dan `type`), pengujian utama dilakukan secara statis oleh compiler TypeScript. Tujuannya adalah untuk memastikan bahwa tipe-tipe ini digunakan secara konsisten di seluruh basis kode tanpa menimbulkan error kompilasi.
  - **Tidak ada pengujian runtime** yang diperlukan untuk file definisi tipe itu sendiri.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| Seluruh Proyek | Kompilasi TypeScript (`tsc`) | Kompilasi berhasil tanpa error terkait tipe dari file-file ini | `Lulus` |

*Catatan: Validasi utama terjadi selama proses build dan di dalam IDE, bukan melalui skrip tes khusus.*
