## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi fungsionalitas komponen dan utilitas yang dapat digunakan kembali.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Komponen UI (`shadcn/ui`):** Pengujian minimal dilakukan karena komponen ini berasal dari library pihak ketiga yang sudah teruji. Fokusnya pada verifikasi kustomisasi jika ada.
  - **Utilitas (`utils.ts`):** Menguji setiap fungsi pembantu dengan berbagai input untuk memastikan output yang benar.
  - **Komponen Umum:** Menguji `VariableListManager` secara menyeluruh karena memiliki logika state yang kompleks untuk memindahkan item antar daftar.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `cn` (di `utils.ts`) | `cn("bg-red-500", false && "bg-blue-500")` | Mengembalikan `"bg-red-500"` | `Lulus` |
| `iconHelper` | Render dengan prop `iconName="Save"` | Ikon 'Save' (SVG) ditampilkan | `Lulus` |
| `VariableListManager` | Pindahkan item dari daftar sumber ke target | Item muncul di daftar target dan hilang dari daftar sumber | `Lulus` |
| `VariableListManager` | Pindahkan semua item ke target | Daftar sumber menjadi kosong | `Lulus` |

*Catatan: Pengujian memastikan bahwa blok bangunan fundamental aplikasi ini andal dan dapat digunakan kembali di berbagai fitur.*
