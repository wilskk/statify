## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap fungsi (actions) dalam store memanipulasi state sesuai harapan.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `Zustand Test Utils`
- **Fokus Pengujian:**
  - **Pengujian Aksi (Actions):** Memvalidasi bahwa `setTableRef` dan `setVariableTableRef` berhasil menyimpan objek referensi ke dalam state.
  - **Pengujian Selektor (Selectors):** Memastikan `getTableRef` dan `getVariableTableRef` mengembalikan objek referensi yang benar dari state.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useTableRefStore` | Panggil `setTableRef` dengan objek mock | State `tableRef` berisi objek mock | `Lulus` |
| `useTableRefStore` | Panggil `setVariableTableRef` dengan objek mock | State `variableTableRef` berisi objek mock | `Lulus` |
| `useTableRefStore` | Panggil `getTableRef` setelah `setTableRef` | Mengembalikan objek mock yang sama | `Lulus` |

*Catatan: Pengujian dilakukan dengan memanipulasi store di lingkungan tes terisolasi dan menegaskan perubahan state.*
