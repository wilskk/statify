Dokumen ini berisi detail pembangunan dan pengujian untuk Feature Set `Tampilan Data`.

## 1. Work Package

- **Fitur yang Dikerjakan:**
  - Menyajikan halaman untuk menampilkan data dalam grid (Data View)
  - Menampilkan header kolom dengan nama variabel dan ikon tipe data
  - Mendukung dua mode tampilan: nilai numerik dan label nilai
  - Menyorot baris yang tidak aktif berdasarkan variabel filter
  - Mengizinkan pengeditan sel dan menyimpan perubahan secara otomatis (debounced)
  - Menyediakan menu konteks untuk berbagai operasi (baris, kolom, clipboard)

## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Pengujian Fungsionalitas (Hooks):** Memvalidasi logika dalam custom hooks, seperti `useTableUpdates` yang menangani perubahan sel (debouncing) dan `useContextMenuLogic` yang menentukan item menu konteks yang benar.
  - **Pengujian Render:** Memastikan komponen `DataGrid` me-render jumlah baris dan kolom yang benar berdasarkan data dari store.
  - **Pengujian Interaksi:** Memvalidasi bahwa interaksi pengguna pada grid (misalnya, mengedit sel) memicu hook yang sesuai.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useTableUpdates` | Panggil `handleCellChange` | Fungsi `updateCellData` di `DataService` dipanggil setelah delay | `Lulus` |
| `useContextMenuLogic` | Klik kanan pada header kolom | Menu konteks untuk kolom ditampilkan | `Lulus` |
| `DataGrid` | Render dengan 10 baris data | Tabel dirender dengan 10 baris `<tr>` | `Lulus` |
| `DataGrid` | Edit sel | Nilai dalam sel diperbarui di UI | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` atau `*.test.ts` yang bersangkutan.*
