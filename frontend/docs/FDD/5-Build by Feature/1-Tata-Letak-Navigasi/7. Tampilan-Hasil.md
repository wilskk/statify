## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`, `Zustand Test Utils`
- **Fokus Pengujian:**
  - **Pengujian Store:** Memvalidasi `useResultStore` dapat menambah hasil baru, menyimpan daftar hasil, dan mengatur hasil yang aktif.
  - **Pengujian Render:** Memastikan `Sidebar` me-render item navigasi sesuai jumlah hasil di store. Memastikan `ResultOutput` me-render komponen yang benar (tabel, editor) berdasarkan tipe konten hasil yang aktif.
  - **Pengujian Interaksi:** Memvalidasi bahwa mengklik item di `Sidebar` akan mengubah hasil yang aktif di `useResultStore`.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useResultStore` | Panggil `addResult` | Hasil baru ditambahkan ke array `results` | `Lulus` |
| `useResultStore` | Panggil `setActiveResult` | `activeResultId` di state diperbarui | `Lulus` |
| `Sidebar` | Store memiliki 2 hasil | Dua item navigasi di-render | `Lulus` |
| `ResultOutput` | Hasil aktif memiliki tipe 'table' | Komponen `DataTable` di-render | `Lulus` |
| `ResultOutput` | Hasil aktif memiliki tipe 'editor' | Komponen `TiptapEditor` di-render | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` atau `*.test.ts` yang bersangkutan.*
