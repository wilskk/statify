## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Pengujian Render:** Memastikan setiap komponen layout (`Header`, `Navbar`, `Toolbar`, `Footer`) berhasil di-render tanpa error saat `DashboardLayout` dimuat.
  - **Pengujian Fungsionalitas:** Memvalidasi bahwa komponen-komponen interaktif seperti menu di `Navbar` atau tombol di `Toolbar` dapat merespons aksi pengguna (meskipun implementasi aksi penuh ada di fitur lain).

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `DashboardLayout` | Render awal | Semua komponen children (Header, Navbar, dll.) termuat | `Lulus` |
| `Header` | Tampilan judul aplikasi | Judul aplikasi tampil dengan benar | `Lulus` |
| `Navbar` | Render menu 'File' | Menu 'File' beserta item-itemnya tampil | `Lulus` |
| `Toolbar` | Render tombol 'Simpan' | Tombol 'Simpan' tampil dan aktif | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` yang bersangkutan.*
