## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Pengujian Render:** Memastikan setiap komponen konten (`GettingStarted`, `FAQ`, `Feedback`) berhasil di-render di dalam `HelpPage`.
  - **Pengujian Interaksi:** Memvalidasi fungsionalitas UI seperti membuka/menutup item accordion pada `FAQ`.
  - **Pengujian Fungsionalitas:** Menguji validasi dan pengiriman formulir pada komponen `Feedback`.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `HelpPage` | Render awal | Halaman me-render semua komponen konten tanpa crash | `Lulus` |
| `GettingStarted` | Tampilan konten | Teks panduan memulai terlihat oleh pengguna | `Lulus` |
| `FAQ` | Klik pada pertanyaan | Jawaban untuk pertanyaan tersebut muncul | `Lulus` |
| `Feedback` | Submit formulir kosong | Pesan error validasi muncul | `Lulus` |
| `Feedback` | Submit formulir valid | Pesan sukses muncul dan form di-reset | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` yang bersangkutan.*
