## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`, `Zustand Test Utils`
- **Fokus Pengujian:**
  - **Pengujian Store:** Memastikan state management untuk modal (`useModalStore`) berfungsi dengan benar (membuka, menutup, mengatur props).
  - **Pengujian Fungsionalitas:** Memvalidasi bahwa `ModalManager` dapat mengambil komponen yang benar dari `ModalRegistry` dan me-render-nya melalui `ModalRenderer` ketika state di store berubah.
  - **Pengujian Render:** Memastikan tidak ada modal yang dirender saat `activeModal` bernilai `null`.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `useModalStore` | Panggil `openModal('TestModal', {prop: 1})` | `activeModal` menjadi 'TestModal' dan `modalProps` berisi `{prop: 1}` | `Lulus` |
| `useModalStore` | Panggil `closeModal()` | `activeModal` menjadi `null` dan `modalProps` menjadi `{}` | `Lulus` |
| `ModalManager` | State store berubah ke `activeModal: 'TestModal'` | Komponen `TestModal` di-render | `Lulus` |
| `ModalManager` | State store berubah ke `activeModal: null` | Tidak ada modal yang di-render | `Lulus` |
| `ModalRegistry` | Akses registry | Mengembalikan komponen yang terdaftar dengan benar | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` atau `*.test.ts` yang bersangkutan.*
