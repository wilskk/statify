## 3. Pengujian Unit (Unit Testing)

Pengujian unit dilakukan untuk memverifikasi bahwa setiap komponen dan fungsi bekerja sesuai harapan secara terisolasi.

### 3.1. Strategi Pengujian

- **Framework:** `Jest`, `React Testing Library`
- **Fokus Pengujian:**
  - **Pengujian Render:** Memastikan setiap seksi (`Hero`, `Features`, `Comparison`, `Testimonials`) berhasil di-render tanpa error.
  - **Pengujian Tampilan Konten:** Memvalidasi bahwa konten statis seperti teks judul dan gambar tampil dengan benar di setiap seksi.

### 3.2. Skenario & Hasil Pengujian

| Komponen/Hook/Service | Skenario Pengujian | Hasil yang Diharapkan | Status |
| :-------------------- | :----------------- | :-------------------- | :----- |
| `LandingPage` | Render awal | Halaman me-render semua seksi tanpa crash | `Lulus` |
| `HeroSection` | Tampilan `headline` | Teks `headline` utama terlihat oleh pengguna | `Lulus` |
| `FeaturesSection` | Tampilan daftar fitur | Daftar fitur aplikasi (minimal 3) tampil | `Lulus` |
| `ComparisonSection` | Tampilan tabel perbandingan | Tabel perbandingan dengan kompetitor tampil | `Lulus` |
| `TestimonialsSection` | Tampilan testimoni | Kartu testimoni (minimal 1) tampil | `Lulus` |

*Catatan: Detail skenario pengujian dan implementasi kode tes dapat ditemukan di file `*.test.tsx` yang bersangkutan.*
