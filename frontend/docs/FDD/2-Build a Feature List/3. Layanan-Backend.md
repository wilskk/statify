# Area Fitur Utama: Layanan Backend

Dokumen ini merangkum fitur-fitur yang memberikan nilai bisnis kepada pengguna melalui layanan backend. Setiap fitur dirancang untuk memenuhi kebutuhan pengguna dalam mengelola dan menganalisis data statistik.

> Catatan: Jalur berkas menggunakan alias `@/` sesuai struktur repositori.

---

## 1. Manajemen Berkas Data

| Fitur | Deskripsi | Lokasi Berkas |
| :------------------------------------------------------- | :------------------------------------------------------- | :---------------------------------------------- |
| Unggah berkas data SPSS (.sav) | Pengguna dapat mengunggah berkas data SPSS untuk dianalisis dalam aplikasi | `@/backend/server/services/savService.ts` |
| Unduh hasil analisis dalam format SPSS | Pengguna dapat mengunduh hasil analisis dan data yang telah diproses dalam format SPSS | `@/backend/server/services/savService.ts` |

## 2. Layanan API untuk Analisis Data

| Fitur | Deskripsi | Lokasi Berkas |
| :------------------------------------------------------- | :------------------------------------------------------- | :---------------------------------------------- |
| Endpoint untuk operasi data | Menyediakan API untuk upload, download, dan transformasi data yang dibutuhkan pengguna | `@/backend/server/routes/savRoutes.ts` |
| Pemrosesan permintaan analisis | Menangani permintaan analisis statistik dari pengguna dan mengembalikan hasil yang akurat | `@/backend/server/controllers/savController.ts` |

## 3. Integrasi Frontend-Backend

| Fitur | Deskripsi | Lokasi Berkas |
| :------------------------------------------------------- | :------------------------------------------------------- | :---------------------------------------------- |
| Layanan komunikasi API | Memungkinkan frontend berkomunikasi dengan backend untuk operasi data dan analisis | `@/services/api/savService.ts` |
| Konfigurasi koneksi API | Mengatur koneksi yang stabil dan aman antara frontend dan backend | `@/services/api/config.ts` |

## 4. Keamanan dan Performa

| Fitur | Deskripsi | Lokasi Berkas |
| :------------------------------------------------------- | :------------------------------------------------------- | :---------------------------------------------- |
| Pembatasan akses untuk mencegah overload | Memastikan aplikasi tetap responsif dengan membatasi jumlah permintaan per pengguna | `@/backend/server/app.ts` |
| Keamanan lintas domain | Memungkinkan aplikasi web berjalan dengan aman di berbagai domain | `@/backend/server/app.ts` |
| Perlindungan keamanan web | Melindungi pengguna dari serangan web yang dapat membahayakan data mereka | `@/backend/server/app.ts` |

## 5. Monitoring dan Pemeliharaan

| Fitur | Deskripsi | Lokasi Berkas |
| :------------------------------------------------------- | :------------------------------------------------------- | :---------------------------------------------- |
| Pemeriksaan status layanan | Memastikan layanan backend berjalan dengan baik sehingga pengguna dapat menggunakan aplikasi | `@/backend/server/app.ts` |
| Pembersihan data sementara | Menghemat ruang penyimpanan server untuk memastikan performa optimal aplikasi | `@/backend/server/controllers/savController.ts` |

---

Dengan fitur-fitur di atas, layanan backend memberikan nilai bisnis yang jelas kepada pengguna: kemampuan untuk mengelola data, melakukan analisis statistik, dan mendapatkan hasil yang akurat dengan keamanan dan performa yang optimal.
